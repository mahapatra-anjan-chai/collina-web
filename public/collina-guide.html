How to Use It — Public View

No login required. Works on any mobile browser. Open the link and you're ready to go.

1

Open the app

Go to collina-web.vercel.app on your phone. No account needed.

2

Select players

Tap the names of everyone who showed up today — 14, 15, or 16 players. The grid is sorted by recent appearances so regulars appear first.

3

Generate Teams

Tap "Generate Teams". Collina runs its position-weighted algorithm and produces two balanced teams instantly.

4

Swap players (optional)

Not happy? Tap any player, then tap someone on the other team to swap them. The teams update live.

5

Submit for Manager Approval

Tap "Submit for Manager Approval". Everyone will see a yellow banner on the home screen showing a split has been suggested.

6

After the game — Log Result

Go to /teams → tap "Log Result" → enter the score. The manager reviews and approves it to save it to history.

Home Screen Banners

⏳ A team split has been suggested — teams are pending manager review
✅ Teams are locked in! — teams are final, no more changes possible
2

How to Use It — Manager View

Go to /manager and enter your manager token. The token is verified server-side — wrong tokens are rejected with a 401.

🔒

Lock & Publish Teams

Review the player-submitted split and lock it in as final. Green banner shows to everyone.

↩

Revert to Original

If players manually swapped positions, revert back to the algorithm's original output before publishing.

⚽

Generate New Teams (with stats)

Pick 16 players and generate a fresh split yourself. Sees full OVR + Composite scores hidden from public.

🗑

Reset Today's Teams

Clears suggested + official teams so players can generate from scratch. Post-game history is NOT affected.

✅

Approve Post-Game Result

Review a pending result submitted by players. Add official notes, then publish to history permanently.

🗑

Discard Post-Game Result

If the result is wrong or spam, discard it without saving anything to history.

Retroactive Game Logging

If teams were reset before a result was logged, the manager can submit a full game record directly via the approve API endpoint — no pending result required. The endpoint accepts teamA, teamB, scoreA, scoreB, and date directly in the body.

The Balancing Algorithm

Each player gets a composite score computed using position-specific skill weights — so a keeper's Defending matters more than their Shooting, and a striker's Pace and Shooting dominate.

Position Weight Matrix

Position Dribbling & Passing Defending Shooting Pace
Keeper 0.05 0.50 0.00 0.45
Defender 0.10 0.45 0.05 0.40
Mid 0.40 0.20 0.25 0.15
Striker 0.25 0.05 0.40 0.30

Algorithm Steps

Load Weight Adjustments

Check for per-player skill modifiers from past post-game feedback (stored in KV). Apply multipliers before computing scores.

Compute Composite Score

For each player: Composite = (D&P × w₁) + (Def × w₂) + (Shoot × w₃) + (Pace × w₄) using their position weights above.

Defending Override Rule

If a Defender's overall is below median but their Defending is above the 75th percentile, their Defending weight is boosted by 1.25× and the 4 weights are renormalised to sum to 1.0.

Group by Position + Serpentine Draft

Players are grouped into 4 buckets (Keeper/Defender/Mid/Striker), sorted by composite score, then allocated in a serpentine (snake) draft. Starting team alternates between position groups.

Global Balance Check

Mean OVR difference must be ≤ 0.5. Std Dev difference must be ≤ 0.3. If thresholds exceeded, up to 3 same-position swaps are attempted to fix the imbalance.

Independent Verification

A separate verification agent re-runs all composite score calculations and re-checks balance metrics. If it fails, the swap phase runs again for up to 2 loops total.

API Routes

MethodEndpointAuthDescription
POST/api/generateNoneRun balancing algorithm for selected players
GET/api/suggestedNoneGet latest suggested teams
POST/api/suggestedNoneSave player-submitted teams to KV
GET/api/officialNoneGet locked official teams (names only, no stats)
POST/api/publishManagerLock & publish suggested teams as official
POST/api/revertManagerRevert suggested back to original algorithm output
POST/api/resetManagerClear all team state (preserves history)
GET/api/historyNoneAll game history, last 12 months, newest first
DELETE/api/historyManagerDeduplicate history entries (same date + result)
GET/api/postgameNoneGet pending result + latest approved result
POST/api/postgameNoneSubmit game result (requires teams to be locked)
DELETE/api/postgameManagerDiscard pending result without saving
POST/api/postgame/approveManagerApprove pending result → history. Supports direct-write mode for retroactive logging.
GET/api/authManagerVerify manager token — returns 200 or 401
GET/api/adjustmentsNoneGet active player weight adjustments
